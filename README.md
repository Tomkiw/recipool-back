# BackEnd

Технічне завдання серверної частини проєкту **Recipes** (`recipool-back`) —
REST API для пошуку, створення та збереження кулінарних рецептів.

## Загальні завдання

1. Розгорнути сервер для розробки (підключити модулі, налаштувати CORS, написати функцію для вилову помилок тощо).
2. Обговорити структуру, ініціалізувати та підключити БД до проєкту.
3. Створити swagger-документацію енд-поінтів.

## Ендпоінти

### `/auth`

- Публічний ендпоінт реєстрації користувача.
- Публічний ендпоінт логінізації користувача.
- Прошарок авторизації (middleware).
- Приватний ендпоінт для логаута користувача.

### `/users`

- Приватний ендпоінт на отримання інформації про поточного користувача.

### `/api/categories`

- Публічний ендпоінт для отримання списку категорій рецептів.

### `/api/ingredients`

- Публічний ендпоінт для отримання списку інгредієнтів.

### `/api/recipes`

- Публічний ендпоінт для пошуку рецептів за категорією, інгредієнтом та входженням пошукового значення в назву рецепту (з урахуванням логіки пагінації).
- Публічний ендпоінт для отримання детальної інформації про рецепт за його `id`.
- Приватний ендпоінт для створення власного рецепту.
- Приватний ендпоінт для отримання власних рецептів.
- Приватний ендпоінт для додавання рецепту до списку улюблених.
- Приватний ендпоінт для видалення рецепту зі списку улюблених.
- Приватний ендпоінт для отримання улюблених рецептів.

## Додаткові завдання

Виконуються за умови, що було реалізовано всі основні.

### `/api/recipes`

- Приватний ендпоінт для видалення власного рецепту. Продумати логіку для такого рецепту у випадку, якщо його було додано в список улюблених іншими користувачами.

## Реалізовані маршрути

| Метод    | Шлях                             | Доступ    | Опис                                                        |
| -------- | -------------------------------- | --------- | ----------------------------------------------------------- |
| `POST`   | `/auth/register`                 | Публічний | Реєстрація користувача                                       |
| `POST`   | `/auth/login`                    | Публічний | Логінізація користувача                                      |
| `POST`   | `/auth/refresh`                  | Публічний | Оновлення сесії користувача                                  |
| `GET`    | `/auth/session`                  | Приватний | Отримання поточної сесії                                     |
| `POST`   | `/auth/logout`                   | Приватний | Логаут користувача                                           |
| `GET`    | `/users/current`                 | Приватний | Інформація про поточного користувача                        |
| `PATCH`  | `/users/avatar`                  | Приватний | Оновлення аватара користувача                               |
| `GET`    | `/api/categories`                | Публічний | Список категорій рецептів                                   |
| `GET`    | `/api/ingredients`               | Публічний | Список інгредієнтів                                         |
| `GET`    | `/api/recipes`                   | Публічний | Пошук рецептів за категорією, інгредієнтом, назвою + пагінація |
| `GET`    | `/api/recipes/:recipeId`         | Публічний | Детальна інформація про рецепт за `id`                      |
| `POST`   | `/api/recipes`                   | Приватний | Створення власного рецепту                                  |
| `GET`    | `/api/my/recipes`                | Приватний | Отримання власних рецептів                                  |
| `GET`    | `/api/recipes/favorites`         | Приватний | Отримання улюблених рецептів                                |
| `POST`   | `/api/recipes/:recipeId/favorite`| Приватний | Додавання рецепту до улюблених                              |
| `DELETE` | `/api/recipes/:recipeId/favorite`| Приватний | Видалення рецепту з улюблених                               |

## Технології

- **Node.js** + **Express 5**
- **MongoDB** + **Mongoose**
- Сесійна авторизація (access/refresh токени в httpOnly-cookie) + `bcrypt`
- **celebrate** (Joi) для валідації запитів
- **multer** + **Cloudinary** для завантаження зображень
- **pino-http** для логування
- **swagger-ui-express** для документації API

## Запуск проєкту

```bash
# Встановлення залежностей
npm install

# Режим розробки (з автоперезапуском)
npm run dev

# Продакшн-запуск
npm start

# Генерація swagger-документації
npm run swagger
```

Документація API доступна після запуску за адресою `/api-docs`.

## Змінні оточення

Створіть файл `.env` у корені проєкту:

```env
PORT=3000
MONGO_URL=<рядок підключення до MongoDB>
FRONTEND_URL=<дозволені origin для CORS, через кому>

CLOUDINARY_CLOUD_NAME=<...>
CLOUDINARY_API_KEY=<...>
CLOUDINARY_API_SECRET=<...>
```
